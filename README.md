# Small Variants Simplification Pipeline

A Python tool to simplify small variants in pangenome graphs in GFA1.1 format while preserving the information required to reconstruct path offsets and approximate sequences.

The pipeline is divided into two steps:

1. **simplify**: simplifies small variants in a pangenome graph and produces a simplified graph together with the auxiliary files required for offset computation.
2. **offsets**: computes and adds positions offsets tags to a graph. Supports simplified graphs using the auxiliary files produced during the simplification step.

The two commands are independent: if the input graph has not been simplified, the `offsets` command can still be executed, although it will simply reproduce the original path offsets.

# Requirements

## Software

- Python >= 3.10
- `vg` (must be installed and available in the system `PATH`)

## Python dependencies

The project depends on:

- `gfagraphs`
- `networkx`
- `matplotlib`

All required Python dependencies are automatically installed during the installation process.

# Installation

The recommended installation method uses **uv**, a fast Python package and virtual environment manager. If you already use another environment manager (such as `venv`, `conda` or `mamba`), you can use it instead, as long as you install the project dependencies.

Clone the repository:

```bash
git clone git@github.com:AliceLaurent/pgsimplify.git
cd pgsimplify
```

Create and activate a virtual environment:

```bash
uv venv
source .venv/bin/activate
```

Install the dependencies using the provided lock file:

```bash
uv pip sync uv.lock
```

Then install the package in editable mode:

```bash
uv pip install -e .
```

You can verify that the installation was successful with:

```bash
pgsimplify --help
```

# Usage

## Simplify

Simplify small variants in a pangenome graph.

```bash
pgsimplify simplify <input_gfa> <output_dir> [options]
```

### Required arguments

| Argument | Description |
|----------|-------------|
| `input_gfa` | Input GFA graph. |
| `output_dir` | Directory where all output files will be written. |

### Optional arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--max-len-to-collapse` | `50` | Maximum MNP length to collapse. |
| `--min-variant-size` | `50` | Variants smaller than this threshold are simplified. |
| `--no-subgraphs-save` | disabled | Do not export subgraphs generated during simplification. |

Example:

```bash
pgsimplify simplify graph.gfa results/
```

or

```bash
pgsimplify simplify graph.gfa results/ \
    --max-len-to-collapse 100 \
    --min-variant-size 75
```

## Offset

Compute paths offsets tags from a graph in GFA1.1 format.

The command supports simplified graphs generated by the `simplify` command.

```bash
pgsimplify offset <input_dir> <output_gfa>
```

- **subgraphs/** contains one GFA file for each exported subgraph (unless `--no-subgraphs-save` is specified).

Temporary files required by `vg` are created automatically during execution and removed afterwards.

---

# Pipeline overview

The simplification pipeline consists of two main stages.

1. Graph compression.
2. Graph simplification based on the snarl decomposition computed by `vg`.

Snarls are computed internally using the `vg` toolkit. Intermediate files required for the conversion between GFA and VG formats are handled automatically.

---

# Limitations

- The pipeline currently relies on the `vg` toolkit to compute snarls.
- The expected directory structure for the `offset` command must be preserved.
